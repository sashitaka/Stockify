
# Rapport Stockify

### Diagrammes

Diagramme de classe

Classe user qui represente un utilisateur avec comme attributs ses informations de compte et une liste de stock du type PortfolioItem. PortfolioItem represente une stock que le user possede, cette classe herite de la classe stock. La classe stock represente une stock typique. Classe transactions represente une transaction effectuer par le user vente ou achat d'une stock contenant les infos pertinentes de la transac. La classe admin sera un user qui pour gerer l'application notamment supprimer un user client, d'autres actions seront defini au fur et a mesure
